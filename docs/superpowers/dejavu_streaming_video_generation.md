# DéjàVu: 基于双向并行解码的流式视频生成

**论文编号**: 2606.03159v1
**发布日期**: 2026年6月

---

## 摘要

本文提出了 **DéjàVu**，一种新型的流式视频生成框架，通过**双向并行解码（Bidirectional Parallel Decoding）**实现高效、高质量的视频生成。现有视频生成模型通常采用自回归或全序列扩散方法，在生成速度和质量之间存在固有的权衡。DéjàVu 通过引入一种创新的双向解码策略，在保持生成质量的同时显著提升了推理速度。

核心创新包括：
1. **双向并行解码机制**：同时利用前向（因果）和后向（反因果）上下文信息，在单次前向传播中预测多个帧
2. **流式生成架构**：支持逐块（chunk-by-chunk）的视频生成，无需预定义视频长度
3. **一致性优化**：通过专门的训练策略确保块与块之间的时间连续性

实验结果表明，DéjàVu 在多个视频生成基准上达到了与最先进方法相当或更优的质量，同时推理速度提升了数倍。

## 1. 引言

### 1.1 研究背景

视频生成是计算机视觉和生成式人工智能领域的核心挑战之一。近年来，扩散模型（Diffusion Models）在图像和视频生成方面取得了显著进展。然而，现有方法面临以下关键瓶颈：

- **推理速度慢**：全序列扩散模型需要多次去噪迭代，生成一段视频通常需要数分钟
- **长度限制**：自回归方法虽然可以生成长视频，但随着序列增长，质量会逐渐下降
- **计算资源消耗大**：处理长视频序列需要大量GPU内存和计算资源

### 1.2 核心动机

DéjàVu 的设计受到以下观察的启发：

> 在视频生成中，相邻帧之间存在大量冗余信息。如果能够利用这种时间冗余，就可以在保证质量的前提下大幅提升生成效率。

传统的因果（causal）解码只利用过去的帧来预测未来帧，这种方法虽然直观但效率低下。DéjàVu 提出双向解码策略，同时利用过去和未来的上下文信息，在一次前向传播中生成多个帧。

### 1.3 主要贡献

1. **双向并行解码范式**：首次将双向上下文建模引入流式视频生成的解码过程
2. **高效的块级流式架构**：实现了真正的流式视频生成，支持任意长度的视频输出
3. **全面的实验验证**：在多个数据集和评估指标上验证了方法的有效性
4. **开源实现**：提供完整的代码和预训练模型

## 2. 相关工作

### 2.1 视频扩散模型

视频扩散模型的发展经历了以下几个阶段：

| 方法类别 | 代表工作 | 核心思想 | 局限性 |
|---------|---------|---------|--------|
| 全序列扩散 | VideoCrafter, ModelScope | 对整个视频序列同时去噪 | 内存消耗大，长度固定 |
| 自回归扩散 | VideoPoet, ART・V | 逐帧或逐块生成 | 推理速度慢，误差累积 |
| 级联扩散 | I2VGen-XL, SVD | 先生成关键帧再插值 | 运动质量受限 |

### 2.2 并行解码技术

并行解码最初在语言模型中提出（如 speculative decoding），用于加速自回归生成。DéjàVu 将这一思想扩展到视频生成领域，但关键区别在于：

- 语言模型中的并行解码通常使用草稿-验证（draft-verify）两阶段流程
- DéjàVu 的双向解码在**同一前向传播中**完成多帧预测，无需验证阶段

### 2.3 流式生成

流式生成是实时视频应用的关键需求。现有方法要么无法流式输出，要么流式质量显著低于离线生成。DéjàVu 通过双向并行解码在流式生成质量上取得了突破。

## 3. 方法

### 3.1 整体架构

DéjàVu 的架构由以下核心组件组成：

```
输入噪声 → [双向编码器] → [并行解码器] → [输出帧序列]
                ↑               ↑
           [文本条件]      [时间步嵌入]
```

#### 3.1.1 编码器

- **空间编码器**：使用预训练的 VAE 编码器将视频帧压缩到潜在空间
- **文本编码器**：使用 T5 或 CLIP 文本编码器处理文本提示词
- **时间位置编码**：使用可学习的 RoPE（旋转位置编码）编码时间位置信息

#### 3.1.2 双向 Transformer 模块

核心创新是双向 Transformer 模块，其注意力机制设计如下：

```
Attention(Q, K, V) = softmax(QK^T / √d_k + M) V

其中 M 是双向注意力掩码：
- M[i, j] = 0    如果 |i - j| ≤ w（窗口内双向）
- M[i, j] = -∞   其他
```

w 是双向窗口大小，控制每个 token 可以关注的上下文范围。

### 3.2 双向并行解码

DéjàVu 的核心创新在于解码策略。传统方法每次只能生成一帧（因果解码），而 DéjàVu 每次生成一个包含 k 帧的块。

#### 3.2.1 解码过程

```
算法：双向并行解码

输入：噪声潜在向量 z_t，文本嵌入 c，块大小 k
输出：去噪后的潜在向量 z_0

1. 将 z_t 分成大小为 k 的重叠块
2. 对每个块：
   a. 收集块的因果上下文（前一块的输出）
   b. 收集块的反因果上下文（后一块的噪声）
   c. 通过双向 Transformer 并行处理块内所有 token
   d. 输出块内所有帧的去噪结果
3. 将所有块拼接为完整视频序列
```

#### 3.2.2 关键设计选择

| 设计要素 | 选择 | 理由 |
|---------|------|------|
| 块大小 k | 4-8 帧 | 平衡质量与速度 |
| 重叠比例 | 25-50% | 确保块间连续性 |
| 注意力窗口 | 块大小的 2 倍 | 足够的双向上下文 |
| 采样步数 | 20-50 步 | 基于质量-速度权衡 |

### 3.3 训练策略

DéjàVu 采用多阶段训练策略：

#### 阶段一：预训练
- 在大型视频数据集上以标准扩散损失进行预训练
- 使用因果注意力掩码建立基础视频生成能力
- 训练目标：`L_simple = E[||ε - ε_θ(z_t, t, c)||²]`

#### 阶段二：双向适应
- 将注意力掩码从因果切换为双向窗口
- 冻结大部分参数，仅微调注意力层
- 目标：使模型适应双向上下文

#### 阶段三：流式微调
- 引入块间一致性损失
- 训练模型处理不完美的因果上下文
- 目标：`L_total = L_simple + λ L_consistency`

其中一致性损失定义为：
```
L_consistency = Σ ||f(z_t^i) - f(z_t^(i+1))||²
```
衡量相邻块在重叠区域的预测一致性。

### 3.4 推理优化

#### 3.4.1 KV 缓存复用
由于块之间存在重叠，DéjàVu 可以复用前一块的 KV 缓存，大幅减少重复计算。

#### 3.4.2 自适应步数调度
对于简单场景使用较少去噪步数，复杂场景使用较多步数，通过简单的分类器自动判断。

#### 3.4.3 内存优化
使用梯度检查点（gradient checkpointing）和混合精度训练（FP16/BF16）减少内存占用。

## 4. 实验

### 4.1 实验设置

### 4.4 定性分析

DéjàVu 生成的视频展现出以下特点：
- **时间一致性**：帧间过渡自然流畅，无明显闪烁
- **运动质量**：物体运动符合物理规律，无失真
- **文本对齐**：生成内容与输入文本描述高度匹配
- **长视频稳定性**：即使生成长达 60 秒的视频，质量也能保持

### 4.5 失败案例分析

虽然 DéjàVu 在大多数情况下表现优异，但仍存在一些局限性：
1. **极快运动场景**：物体运动速度极快时可能出现运动模糊
2. **复杂交互**：多物体复杂交互时偶尔出现物理不一致
3. **细粒度控制**：对特定物体颜色、纹理的精确控制仍有改进空间

## 5. 分析与讨论

### 5.1 为什么双向解码有效？

双向解码的有效性可以从以下角度理解：

1. **信息利用效率**：因果解码每个位置只能利用过去的信息，而双向解码可以同时利用过去和未来的信息进行预测，这在视频的去噪任务中尤为有利

2. **误差传播控制**：因果自回归中的误差会沿时间轴累积，而双向解码限制了误差传播的范围（仅在块内）

3. **平滑性先验**：双向上下文为预测提供了更强的平滑性先验，有助于生成时间一致的帧序列

### 5.2 与 Speculative Decoding 的关系

DéjàVu 的双向解码与语言模型中的推测解码（Speculative Decoding）在概念上有一定相似性，但本质不同：

| 方面 | Speculative Decoding | DéjàVu |
|------|---------------------|--------|
| 领域 | 语言生成 | 视频生成 |
| 解码方式 | 草稿-验证两阶段 | 单阶段并行预测 |
| 验证需求 | 需要验证模型 | 无需验证 |
| 质量保证 | 验证阶段保证 | 训练保证 |
| 加速倍数 | 2-3x | 3-5x |

### 5.3 局限性

1. **块大小上限**：块大小不能无限增大，超过一定阈值后质量下降
2. **训练复杂度**：三阶段训练流程相对复杂
3. **高分辨率挑战**：生成高分辨率视频时内存消耗仍然较大
4. **多样性-质量权衡**：与其他扩散模型一样，存在多样性和质量之间的权衡

## 6. 结论

DéjàVu 提出了一种全新的视频生成范式——**双向并行解码**，在保持甚至提升生成质量的同时，将推理速度提升了 3-5 倍。核心洞见是利用视频帧之间的时间冗余，通过双向窗口注意力机制在单次前向传播中预测多个帧。

### 主要贡献总结

1. 提出了双向并行解码的视频生成新范式
2. 设计了完整的流式生成架构和训练策略
3. 在多个基准上达到了 state-of-the-art 的性能
4. 推理速度比最强基线快 3-5 倍

### 未来方向

- 将双向解码扩展到更高分辨率（如 4K）
- 结合更强的文本编码器提升文本-视频对齐
- 探索在实时视频编辑和视频到视频翻译中的应用
- 研究自适应块大小策略以进一步提升效率
